# 土日祝日のやることリストアプリ

## 目的

土日の時間を有効活用したい(ダラダラ過ごして無駄にしたく無い)

## 機能

- todoの登録
- todoのグループ化
- todoの消化
  - それぞれのtodoの消化ボタンを押すと、設定している時間をカウントダウン
  - 時間が来たらアラームで終了の合図
  - 時間計測ではなく、未/済で消化を判定するtodoもある
    - 時間計測で消化：勉強やアプリ開発、ゲーム等
    - 未/済での消化：家事、ランニング等
  - その日に消化したものは色を変えたりして、一目でわかるようにする
  - 全todoの合計時間を表示する(消化済み時間/消化予定時間)
- グループの利用ログを集計
  - どのtodoをどれくらいやったか
  - 期間指定してみれる

- 日毎に初回アクセスの遷移先を変更する
  - 土日：todo実行画面(土日かつ初回アクセス時に、管理画面への遷移ボタンをつける)
  - 平日：todo管理画面

## テーブル

- タスク登録に利用
  - task_name:string
  - task_timer_mm:string
  - task_group:number
- タスクのログ
  - group_name:string
  - task_name:string
  - execution_time:string
  - extra_time:string
- タスクのグループ
  - group_name:string

## 画面(画面単位に書き出し)

- todo管理
  - todoの表示をリスト形式で行う
  - 表示内容は、「todo名」「todo_実行時間」「土日祝日のタイプ(消化する日を一目でみれるようにする)」「編集ボタン」「削除ボタン」
- todo消化
  - その日の消化すべきtodoが表示される
  - 消化ボタンを押すと、カウントダウン開始
  - カウントダウン終了後、アラームを鳴らし、画面常にボタンを表示してクリックで消化となる
  - 消化しているtodoがある場合には他のタスクは消化できない(消化出来るtodoは逐一1つ。平行して実施は不可)
  - 消化ボタンを押したら、サブウィンドウで大きく「todo名」「カウントダウンタイム」「停止ボタン」を表示する
- todo集計
  - 計測項目
    - 何時間消化したか/消化率(実行時間x分に対して、実際に消化できたy分で計算 y/x)/???
  - 観点
    - todo項目毎/グループ毎/曜日毎
